### D06182605 Versioning and Compatibility Policy

Challenges.md point C19, "Versioning and Compatibility Policy Are Missing", is resolved by adopting a public
compatibility policy in `docs/Compatibility.md` and a development contract in
`docs/specifications/CompatibilityPolicy.md`.

v1 supports Python 3.11 and newer, targets PySpark 3.5.x and 4.0.x by default, avoids a hard Airflow dependency, and
treats Linux as the runtime target. The default generated-code target is `target_pyspark = ">=3.5,<4.1"`.

Structure follows semantic versioning after 1.0. Generated PySpark is committed compiler output and should carry a
compact version header naming the Structure generator version and configured PySpark target. Compiler lineage metadata
has versioned provenance and static dataflow schemas, and config schema changes follow the same compatibility
discipline as other public contracts.

Spark Connect is scheduled for v4. It may land earlier only if it can be supported inside the existing PySpark emitter
boundary without changing public DSL syntax, generated class construction, `run(...)` signatures, streaming
orchestration semantics, or generated-code reviewability.
