### D06192601 Online Execution Default

Structure v1 defaults to online execution. Users construct transform invocations with named input DataFrames and run
them through `StructureSession`:

```python
result = EnrichOrders(
    orders=orders_df,
    customers=customers_df,
).run(session)
```

The transform instance is a deferred invocation. Construction binds inputs and does not start Spark work. The session
owns Spark, optional hook context, resolved configuration, execution mode, target backend, and runner selection.

Generated PySpark remains a first-class optional artifact for provenance, review, debugging, snapshot tests, and
projects configured with `execution_mode = "generated"`. It is no longer the default operational artifact for ordinary
v1 users.

Online execution must consume compiler IR directly and lower it to live PySpark DataFrame and Column operations. It
must not execute generated Python source text. Generated and online execution must preserve the same transform
semantics for supported features, with shared lowering where practical and parity tests where direct sharing would be
premature.

Compiler commands remain Spark-free. `structure check`, `structure compile`, and generated-file diff checks must not
require PySpark, Java, SparkSession, Spark startup, or a Spark cluster. Online execution is a runtime boundary and may
import PySpark.

This decision refines, rather than removes, the generated-code ownership decision. Generated files are still owned by
the compiler when a project chooses to emit and commit them. The default v1 user path simply no longer requires those
files to be part of the end user's repository.
