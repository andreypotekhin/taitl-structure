### D06172605 Keep Schema as the Base Class Name

Keep `Schema` as the canonical base class name instead of renaming it to `Structure`.

`Schema` is the clearest name for the end-user concept: a declared row contract used for Spark `StructType` generation,
validation, lineage, and transform type flow. `Structure` would collide conceptually with the product name, the Python
package name, and the `Struct(...)` nested-field type constructor. It would also make examples such as
`from structure import Structure` read awkwardly and make `class Customer(Structure)` less precise than
`class Customer(Schema)`.

The current spelling already keeps nested schema use natural:

```python
class Address(Schema):
    city = field(String(), nullable=True)


class Customer(Schema):
    address = field(Struct(Address), nullable=True)
```

The framework may later add friendlier aliases only if user research shows real confusion, but v1 documentation and
public API should keep `Schema` canonical.
