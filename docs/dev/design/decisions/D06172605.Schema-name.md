### D06172605 Use Structure as the Base Class Name

Use `Structure` as the canonical base class name instead of `Schema`.

`Structure` aligns the base class with the project name and the end-user authoring mental model without changing the
broader schema terminology used across the compiler, specs, or documentation. The class still represents a declared row
contract used for Spark `StructType` generation, validation, lineage, and transform type flow. The API now prefers
examples such as `from structure import Structure` and `class Customer(Structure)`.

The current spelling already keeps nested schema use natural:

```python
class Address(Structure):
    city = field(String(), nullable=True)


class Customer(Structure):
    address = field(Struct(Address), nullable=True)
```

The framework may later add compatibility aliases if user research shows real friction, but v1 documentation and
public API should keep `Structure` canonical.
