### D06182602 Hook Input Escape Hatch

Challenges.md point C8, "Hook Access to Original Inputs May Need an Escape Hatch", is resolved by keeping the
default hook ABI minimal while adding an explicit opt-in path for hooks that need the original named inputs.

Default hooks keep the signature `def hook(self, *, df, spark, ctx)`. Named input DataFrames are not passed by
default because most hooks only need the current DataFrame, and automatically passing every input would make the hook
contract wider, noisier, and easier to rely on accidentally.

Advanced hooks may request original inputs with `pass_inputs=True` on `@before(...)` or `@after(...)`:

```python
@after(normalize, pass_inputs=True)
def custom_check(self, *, df, inputs, spark, ctx):
    orders = inputs.orders
    return df
```

When `pass_inputs=True` is set, the hook signature must include keyword-only `inputs`. The generated transform passes a
read-only namespace whose attributes match the transform's declared input names, such as `inputs.orders` and
`inputs.customers`. These are the original `run(...)` input DataFrames after normal input schema validation, not the
current intermediate `df`.

Generated code should create and pass this namespace only when at least one hook requests it. This preserves clean,
small generated code for ordinary hooks and keeps hook-free transforms free of source-transform imports and hook
machinery.

Lineage and explain output should mark hooks with `pass_inputs=True` because those hooks may depend on sources that are
not visible through the compiled current-DataFrame path. The hook remains an opaque arbitrary-PySpark boundary; this
escape hatch improves expressiveness without pretending the compiler can reason about the custom code inside it.
