### D06172603 Schema Inheritance

Schema-to-schema inheritance is supported in v1 so developers can reuse common field groups without duplicating schema
code. Effective fields are collected from schema bases in the order written, local fields append last, overrides replace
the inherited field in place, and ambiguous duplicate fields from unrelated bases require an explicit local override.

This balances end-user convenience with deterministic generated schemas. Full semantics are specified in
`docs/dev/design/specifications/SchemaInheritance.spec.md`.
