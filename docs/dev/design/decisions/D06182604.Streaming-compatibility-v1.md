### D06182604 Streaming Compatibility v1

Challenges.md point C11, "Streaming Compatibility Needs a Precise v1 Definition", is resolved by defining streaming
compatibility as a generated DataFrame transform property, not as streaming orchestration. The resolved implementation
contract lives in [StreamingCompatibility.md](../../../specifications/StreamingCompatibility.md).

Structure v1 supports the runtime shape of one streaming current pipeline DataFrame plus zero or more static side
inputs. Generated transforms may be used in a Spark Structured Streaming query when their operations are row-local
projection, row-local filtering, schema-only validation, and stream-static lookup joins. Structure does not generate
`readStream`, `writeStream`, triggers, checkpoints, output modes, query starts, or streaming lifecycle code in v1.

Stream-static `Join.LEFT` and `Join.INNER` joins are allowed when the joined side is static and the join condition
satisfies the join semantics specification. `join_one(...)` uniqueness warnings remain independent of streaming
compatibility. `join_many(...)` (v2) is allowed against static side inputs because row multiplication is intentional
and does not require streaming state by itself; v1 streaming compatibility stays lookup-only until that DSL exists.

Stateful streaming features are deferred. Stream-stream joins, watermarks, aggregations, windowed aggregations,
deduplication, global sorting, limits, Python UDFs, Pandas UDFs, RDD conversion, and Spark actions are batch-only or
outside v1. These may be valid Spark Structured Streaming patterns with additional state and output-mode contracts, but
Structure v1 does not model those contracts.

Hooks are opaque by default. A hook participates in a streaming-compatible transform only when the author explicitly
marks it `streaming_safe=True`, which is a promise that the hook returns a DataFrame and avoids actions, RDD/Pandas
conversion, streaming lifecycle APIs, and stateful streaming operations. The checker records this as a trusted boundary
rather than attempting to analyze arbitrary hook bodies.
