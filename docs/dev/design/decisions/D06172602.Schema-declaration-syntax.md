### D06172602 Schema Declaration Syntax

Challenges.md point C2, "Schema Syntax Needs to Be Finalized", is resolved by adopting explicit type objects as the
canonical v1 schema syntax:

```python
id = field(String(), nullable=False)
total = field(Decimal(12, 2), nullable=True)
ratio = field(Float(), nullable=True)
score = field(Double(), nullable=True)
```

This keeps schema declarations explicit, IDE-friendly, and extensible for nested structs, arrays, generated Spark
`StructType` code, and future adapters. Lowercase type sentinels such as `string`, `decimal(12, 2)`, and `boolean` are
not canonical v1 syntax. Implementation details are specified in
`docs/specifications/SchemaDeclarationSyntax.md`.

Schema-to-schema inheritance is supported separately by
`docs/specifications/SchemaInheritance.md`.
