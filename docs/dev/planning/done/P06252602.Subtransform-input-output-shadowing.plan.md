# Subtransform Input/Output Shadowing

This ExecPlan is complete. It follows `docs/dev/auto/Plans.md`.

## Purpose / Big Picture

Method-level subtransform decorators now use the concise public options `input`, `output`, and `inout`.
The singular `input` and `output` options accept one declaration or an ordered list, so the retired plural
`inputs` and `outputs` options are no longer needed. Method-level `lane` and `lanes` are also retired; hook decorators
still preserve `lane` and `lanes`.

The user-visible API is:

    @transform(input=orders, output=normalized)
    @transform(input=[orders, products], output=[accepted, audited])
    @transform(inout=orders | normalized)
    @transform(inout=orders | [accepted, audited])
    @transform(inout=[normalized, orders] | published)

## Surprises & Discoveries

- Observation: Python can support `source | [targets]` and `[sources] | target` through declaration operator overloads,
  but cannot support `[sources] | [targets]` with plain built-in lists.
  Evidence: neither side of list-to-list pipe is a Structure declaration object, so the DSL has no overload hook.

- Observation: Final output contracts should remain separate from working lane names.
  Evidence: generated PySpark can materialize `published = rows.alias(...)` from an implicit lane and still validate the
  declared final output schema.

## Decision Log

- Decision: Keep method-level `input` directional: it accepts `input(...)` and `lane(...)`, not `output(...)`.
  Rationale: Final outputs are public contract declarations, not source selectors.
  Date/Author: 2026-06-25 / Codex and project owner.

- Decision: Keep method-level `output` directional: it accepts `lane(...)` and `output(...)`, not `input(...)`.
  Rationale: Original inputs are read sources, not write targets.
  Date/Author: 2026-06-25 / Codex and project owner.

- Decision: Use source-order shadowing for method-level `input=declared_input`.
  Rationale: Once a lane with the input's name exists, mid-funnel code should read the current lane rather than
  silently restarting from raw input.
  Date/Author: 2026-06-25 / Codex and project owner.

- Decision: Remove the automatic `result.df` alias.
  Rationale: Explicit output declarations make named result access clearer and avoid a second implicit output API.
  Date/Author: 2026-06-25 / Codex and project owner.
