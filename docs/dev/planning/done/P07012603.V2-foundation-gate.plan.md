## Purpose / Big Picture

V2 feature work should start from a stable compiler foundation rather than from the first analytical feature that
happens to be implemented. After this plan is complete, a contributor can add `join_many(...)`, aggregations, windows,
higher-order helpers, optimization directives, and richer explain output without inventing a new operation metadata
shape for each feature family.

The observable result is a small, verified v2 runway. `structure` still remains a PySpark-only v1 runtime, and v2
feature lowering is still deferred, but the compiler can name v2 operation families, report unsupported v2 capability
requirements deterministically, expose cardinality and streaming-support vocabulary, and import the v2 fixture package
without missing public symbols. This plan must not implement analytical feature semantics such as real `group_by(...)`
or `join_many(...)` lowering.

## Decision Log

- Decision: Start v2 with a foundation gate, not with analytical joins.
  Rationale: Sprint 07 assumes operation cardinality, backend capability diagnostics, fixture shape, traceability, and
  explain output can already name row-filtering and row-multiplying operations. Those are Sprint 06 responsibilities.
  Date/Author: 2026-07-01 / Codex

- Decision: Add public v2 placeholder symbols only when needed for fixture importability, and make them fail before
  target lowering.
  Rationale: Importable fixtures let tests and docs refer to v2 examples, but placeholders must not pretend that v2
  behavior is implemented. Unsupported use should route to backend capability diagnostics or a clear compiler error.
  Date/Author: 2026-07-01 / Codex

- Decision: Keep alternative-backend compatibility reporting as scaffolding in this plan, not as real backend support.
  Rationale: P07012601 accepts reserved syntax and honest pending output. This gate may reserve report vocabulary, but
  Polars, DuckDB, Spark SQL, Ibis, Dask, Ray, and Spark Connect lowering remain deferred.
  Date/Author: 2026-07-01 / Codex

- Decision: Treat the v2 generated fixture tree as golden intent, not as current generated output.
  Rationale: It contains examples of future `join_many`, `group_by`, aggregate, higher-order, and cache operations.
  The foundation gate should keep those examples reviewable without requiring the current compiler to regenerate them.
  Date/Author: 2026-07-01 / Codex

- Decision: Store operation capability as compiler IR metadata, not as target-layer `CapabilityRequirement`.
  Rationale: `OperationCapability(group, name)` keeps the compiler backend-neutral. PySpark lowering maps it to
  `CapabilityRequirement` at the target boundary, where unsupported features already fail through `BACKEND-E2402`.
  Date/Author: 2026-07-01 / Codex

- Decision: Classify v1 `join_one(...)` operations as `select_one`.
  Rationale: `join_one(...)` expresses lookup intent: each left row selects at most one right-side row. This remains the
  right operation vocabulary even when the compiler cannot prove uniqueness and emits the existing join warning.
  Date/Author: 2026-07-01 / Codex

## Context and Orientation

The repository root is `C:\Taitl\Code\taitl\taitl-structure`. Structure is a Python library that compiles a small
schema-first DSL into an intermediate representation, or IR. IR means plain Python records that describe transform
semantics before any PySpark code is generated or executed. V1 supports PySpark as the only executable backend.

M7 is the v2 milestone in `docs/dev/project-management/Milestones.md`. M7A is the foundation slice and Sprint 06 is
its sprint charter in `docs/dev/project-management/sprints/Sprint06_V2ScopeAndAnalyticalIR.md`. Sprint 06 asks for v2
operation taxonomy, IR fields for source location, backend capability, cardinality, streaming compatibility, and
traceability, backend capability names, diagnostic anchors, explain placeholders, parity harness placeholders, and a
small v2 fixture package.

The current operation record is `src/structure/app/compiler/ir/model/OperationPlan.py`. `StepPlan` and `OutputPlan`
store ordered tuples of `OperationPlan`, so source-order operation sequencing already exists. The gap is that
`OperationPlan` can name only v1 filters and joins, and it does not carry the metadata that later v2 plans need.

Backend capability checking lives under `src/structure/app/target/capabilities`. `CapabilityRequirement(group, name)`
names a required target feature. `PySparkCapabilities.require(...)` returns success for supported v1 features and raises
`BackendCapabilityError` with `BACKEND-E2402` for unsupported features. Existing tests already prove
`join.join_many` is unsupported in v1.

Streaming compatibility vocabulary already exists under
`src/structure/app/compiler/compileability/streaming_compatibility`. `StreamingSupport` has compatible, batch-only, and
unknown states. This plan may reuse that vocabulary in operation metadata, but it should not implement streaming
orchestration or new streaming source/sink behavior.

The v2 fixture package lives under `res/testing/model/v2`. Source fixtures are examples of desired v2 authoring
syntax. Generated fixtures and traceability files under `res/testing/model/v2/structure_generated` are golden intent
for future renderers, not artifacts that the current compiler can regenerate.

## Idempotence and Recovery

The implementation should be additive and safe to repeat. Re-running tests may create local cache directories; do not
commit cache artifacts. If v2 fixture import tests create `__pycache__` directories under `res/testing/model/v2`, leave
them untracked or clean them only with an explicit, non-destructive command after confirming they are not tracked.

If a public placeholder symbol accidentally executes during import, move the failure to call time. Importability tests
must import modules without side effects beyond Python bytecode caches.

If adding capability metadata to `OperationPlan` breaks existing tests that construct expected operation records,
update those tests only when the new default values preserve existing semantics. Do not remove existing `filter` and
`join` fields in this plan.

If placeholder public exports feel too broad during implementation, prefer a small reserved namespace or documented
placeholder objects, but keep the v2 fixture source importable. Record the decision in the Decision Log before changing
course.

Do not use `git reset --hard` or revert unrelated working-tree changes. The current workspace may already contain
P07012601 and expression-extension edits; work with them.
